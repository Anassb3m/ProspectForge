# Archived planning documents

These files are preserved for historical context only. They describe earlier
V2/V3 planning assumptions, including IT/cyber targeting and signals that are
not part of the current production path.

Do not treat them as current implementation instructions. The source of truth
is the running code plus the root `README.md`, `GUIDE.md`, and `DEPLOY.md`.

- `prospectforge_spec_python.md` — early Python architecture specification
- `logic_specification.txt` — legacy V2 acquisition/scoring logic
- `PROSPECTFORGE_V3_CLIENT_ACQUISITION_REBUILD_MASTER_SPEC.md` — rebuild plan
  retained as a decision record; implementation may intentionally differ
