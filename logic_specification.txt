rospectForge v2.1 — Free Open-Source Ingestion & Enrichment Pipeline
Full logic: Discovery (DECP) → Validation (Sirene + Reacher + theHarvester) → Prioritization → Presentation in the tool
This extends the original v2.0 spec with a production-grade, solo-buildable nightly sourcing engine. Everything stays free (with one important commercial-license caveat on Reacher), leverages real GitHub/open-data projects for efficiency, and is deliberately scoped to the last-mile human advantage (LinkedIn decision-maker confirmation). No paid APIs. No black-box scoring. Full auditability for GDPR/CNIL B2B prospecting.
1. Architecture Principles (Clever & Efficient)

Nightly batch, not real-time scraping: One Polars-powered job (~5-15 min on small VPS) downloads the pre-consolidated DECP Parquet (already cleaned/enhanced by ColinMaudry’s pipeline), filters aggressively for your Elevya-relevant signals, enriches, scores, and upserts.
Company-level deduplication + history: One Prospect row per SIRET. New awards boost urgency and append to award_history (JSONB). No duplicate companies.
Waterfall enrichment (free only): Sirene (official, free) → theHarvester (OSINT bonus) → smart permutation + Reacher SMTP verification. Gaps land in a clean “Needs manual review” queue.
Compliance as first-class: Every prospect gets immutable data_source + informed_at gate before any Sent event can be logged. Partial-diffusion companies are filtered at source.
Human last-mile preserved: Decision-maker name + LinkedIn verification stays manual (ToS + conversion quality). Tool only accelerates the mechanical parts.
Observable & tunable: Every score component is logged. You can tweak weights after 50-100 real outreaches.

2. Data Sources & GitHub Repos (All Free)

DECP consolidated (primary discovery): https://www.data.gouv.fr/fr/datasets/donnees-essentielles-de-la-commande-publique-consolidees-format-tabulaire/ — Parquet + CSV, updated regularly.
Powered by github.com/ColinMaudry/decp-processing (Prefect + Polars + ijson). We consume their published output (smart) instead of re-running their full pipeline.
Sirene enrichment (sector, size, diffusion status, website): Official INSEE API v3 (api.insee.fr). Free public plan (API key). 30 req/min quota — more than enough for nightly filtered batch.
Email verification (core free replacement for Hunter/Dropcontact): github.com/reacherhq/check-if-email-exists (Reacher). Self-hosted Rust binary + HTTP backend via Docker. Does real SMTP RCPT TO checks without sending mail. Rich response (is_reachable, smtp.is_deliverable, has_full_inbox, is_catch_all, syntax/MX details).
Secondary OSINT sweep: github.com/laramies/theHarvester — optional, runs on company domain to surface public emails/naming conventions (useful for SMEs with GitHub/certs/web footprint; limited on pure offline construction firms).
Email candidate generator: Custom ~30-line Python (standard French patterns). No external dep.

Honest Reacher caveat (commercial use): Dual-licensed (AGPL-3.0 vs commercial). For a tool powering a paying business (Elevya pilots), the commercial license is technically required if you keep ProspectForge proprietary. Self-hosting works perfectly; budget for the license if you go to volume or want to stay fully closed-source. Alternative path later: paid waterfall fallback.

def filter_relevant(df: pl.DataFrame, days_back: int = 90) -> pl.DataFrame:
    cutoff = datetime.utcnow() - timedelta(days=days_back)
    # Target for Elevya: IT/cyber/digital transformation tenders French SMEs can win and may need external capacity
    it_cpv_prefixes = ["72", "48", "62"]  # adjust with real CPV analysis
    keywords = ["cybersécurité", "sécurité informatique", "développement logiciel", 
                "système d'information", "transformation numérique", "cloud", "data"]
    
    return (df
        .filter(pl.col("dateAttribution") >= cutoff)
        .filter(
            (pl.col("codeCPV").str.starts_with(it_cpv_prefixes)) |
            pl.col("objetMarche").str.to_lowercase().str.contains("|".join(keywords))
        )
        # Optional: montant > X or multiple winners logic
        .with_columns([
            pl.col("titulaire_siret").str.slice(0, 9).alias("siren")  # if SIRET present
        ])
    )

Step 2: Sirene Enrichment + Compliance Gate
For each unique SIRET/SIREN (batch with httpx + backoff, respect 30/min):
Pythonasync def enrich_sirene(siret: str, api_key: str) -> dict | None:
    url = f"https://api.insee.fr/entreprises/sirene/V3/siret/{siret}"
    headers = {"X-INSEE-Api-Key-Integration": api_key}
    async with httpx.AsyncClient() as client:
        r = await client.get(url, headers=headers, timeout=10)
        if r.status_code == 200:
            data = r.json()
            ul = data.get("uniteLegale", {})
            etab = data.get("etablissement", {})
            if ul.get("statutDiffusionUniteLegale") == "partiellement_diffusible":
                return None  # hard filter — do not prospect
            if etab.get("etatAdministratif") != "A":
                return None
            return {
                "company_name": ul.get("denominationUniteLegale") or etab.get("denominationUsuelleEtablissement"),
                "naf_code": ul.get("activitePrincipaleUniteLegale"),
                "tranche_effectifs": ul.get("trancheEffectifs") or etab.get("trancheEffectifsEtablissement"),
                "website": etab.get("siteWeb") or ul.get("siteWeb"),  # if present
                "diffusion_status": ul.get("statutDiffusionUniteLegale"),
                # map tranche to your buckets: 11-50, 51-200 etc.
            }
    return None

Step 3: Email Discovery & Validation Waterfall (per prospect or on-demand)
Trigger: nightly for new prospects, or user button “Run contact discovery” in UI.

If website: run theHarvester (subprocess or Python import) → extract any public emails + naming patterns.
Generate candidates (smart permutations):
Common French professional patterns: {prenom}.{nom}, {p}.{nom}, {prenom}{nom}, {nom}.{prenom}, role-based (contact, commercial, direction, marchespublics, info, tenders).
If theHarvester gave a pattern or name hint → weight those first.

Verify batch with Reacher (local http://reacher:8080/v0/check_email or updated endpoint):
Only keep is_reachable == "safe" or strong smtp.is_deliverable.
Store with contact_source="reacher", contact_confidence="verified", full response JSON for audit.

Anything left → “Needs manual review” queue (filterable view). User pastes LinkedIn name → system re-runs permutations + verify instantly.

Result stored:

email (best verified)
decision_maker_name / title (initially null or from user)
phone (Sirene switchboard if present + manual)
linkedin_url (user-filled)

7. Non-Functional & Honest Limits

Performance: One small VPS handles it. Parquet download + Polars filter is fast; Sirene calls are the bottleneck (rate-limited but low volume after filtering).
Cost: €0 (except optional Reacher commercial license + VPS SMTP unblock if needed).
Limits you accept:
No auto decision-maker name (free data doesn’t reliably give the right person for tenders).
Reacher coverage good but not 100% (especially catch-all or strict corporate servers) → manual queue is feature, not bug.
theHarvester shines on companies with public digital footprint.
Phone remains mostly manual/switchboard.

GDPR/CNIL: All data is public awards + registry. Still document data_source rigorously. Never use partial-diffusion records. informed_at + first-contact disclosure best practice.

This pipeline turns “spend 2 days browsing BOAMP” into “open tool every morning → 10-30 high-signal, pre-enriched prospects with verified emails ready for personalized outreach”. The only irreplaceable human step (LinkedIn + first message) is exactly where it creates the most value.
